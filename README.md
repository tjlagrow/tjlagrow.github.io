# tjlagrow.github.io

This repository hosts the academic portfolio website for Theodore J. LaGrow (TJ). It includes the **Ask Waddles** client-side RAG conversational agent, built using WebAssembly ONNX Runtime and custom semantic search algorithms.

## Design Choices: Input & Output Structuring for Small Model Optimization

Running LLMs under 1B parameters (such as SmolLM 135M, Qwen 0.5B, or SmolLM2 360M) entirely within a browser client sandbox requires special input and output structuring. Since client-side execution environment resources are limited, these modifications prevent logical breakdown, style drift, and formatting failures.

### Input Pipeline Structuring

To optimize the retrieval accuracy and semantic alignment before the text is embedded, the system applies the following filters in the input pipeline.

* **Synonym Expansion and Acronym Mapping**: I mapped common abbreviations and synonyms (such as ML, RL, fMRI, and QPP) to their full academic definitions. Smaller models often fail to connect short acronyms with their underlying concepts. This expansion resolves lexical mismatches.
* **Input Length Shortening**: If the user's input exceeds 20 words, the preprocessor strips everything except the first and last sentences. This restricts the input length, preventing long prompts from diluting semantic vector matching weights.
* **Intent Pre-Filtering**: The preprocessor classifies input queries using keyword match boundaries into specific subsets (Teaching, Research, Bio/Contact). This filters the vector database [embeddings.json](file:///c:/Users/tjlag/Documents/tjlagrow.github.io/assets/json/embeddings.json) before similarity scanning, narrowing the search space and saving memory.

### Output Pipeline and Compliance Safeguards

To ensure the output generated by small models remains structured and visually consistent, the system applies the following mechanisms.

* **Dynamic Few-Shot Persona Anchoring**: Small models have difficulty adhering to strict JSON output formats. I developed a style examples database cached in [chatbot.html](file:///c:/Users/tjlag/Documents/tjlagrow.github.io/_includes/chatbot.html). When a user submits a query, the system retrieves the top two most semantically relevant styled JSON examples using cosine similarity and injects them as few-shot prompts. This anchors the model's output formatting and tone.
* **Client-Side Parsing Fallback**: If a small model produces malformed JSON, the parsing helper catches the exception and returns the raw generation, avoiding chatbot crashes.
* **Trailing Sentence Auto-Truncation**: Autoregressive generators under 1B parameters often terminate abruptly in the middle of a sentence when hitting token limits. I integrated a regex-based cleanup filter (`trimToCompleteSentence`) to detect the final punctuation and remove incomplete trailing fragments.
